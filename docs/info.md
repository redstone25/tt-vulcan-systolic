## How it works

This project is a **2ֳ—2 weight-stationary int8 systolic array** (a 4ֳ—4 configuration is verified in simulation; the shuttle build is the 2ֳ—2 silicon config chosen by the area rule) driven by a
tiny 7-instruction ISA (**VULCAN-ISA v0**). It is the silicon half of the
VULCAN mini-EDA/accelerator project; the matching C++ simulator is the golden
model, and this design reproduces its results byte-exactly.

Internals:

- **mac_pe** ג€” one processing element: an int8 weight register (loaded through
  vertical daisy chains), a registered westג†’east activation path, and a 24-bit
  northג†’south partial-sum path (`p_out = p_in + aֲ·w`).
- **systolic_4x4** ג€” the PE grid plus input skew (activation row *k* delayed
  *k* cycles), column-bottom deskew, and a 32-word **int32 accumulating**
  result buffer (accumulation is what makes K-tiling work).
- **vulcan_isa_ctrl** ג€” fetch/decode/execute for the ISA:
  `[31:24] opcode | [23:0] immediate` with `NOP=0 CLR=1 LDW=2 LDA=3 SETO=4
  GEMM=5 HALT=6`, a 16-word instruction buffer and a 256-byte scratch SRAM
  holding packed weights and activations.
- **tt_um_vulcan_systolic** ג€” the byte-bus host interface (see
  `docs/PROTOCOL.md` in this repo): `ui` carries command/data bytes,
  `uio[0]/uio[1]` are edge-detected CMD_VALID/DATA_VALID strobes, `uio[7]/[6]`
  are BUSY/DONE, and `uo` streams results back as little-endian int32.

A GEMM `C(Mֳ—4) = A(Mֳ—K) ֲ· B(Kֳ—4)` is lowered by the host into K/4 tile
passes: `CLR, SETO 0, (LDW, LDA, GEMM M) ֳ— K/4, HALT`, with B tiles and packed
A rows preloaded into the scratch SRAM.

## How to test

The cocotb test (`test/test.py`) replays golden vectors generated by the
VULCAN simulator (`vulcan --isa-vectors`): a 4ֳ—4 ֳ— 4ֳ—2 int8 GEMM. It loads
the 9-instruction program and 24 data bytes over the byte bus, pulses RUN,
polls DONE, streams back all 8 result words, and asserts a byte-exact match
with `test/vectors/expected_c.hex`. A second test checks reset-and-reload.

On real silicon, from the RP2040 firmware:

1. `CMD 0x01` (WR_INSN), then write the program 4 bytes/word little-endian.
   For example, this `.vasm`, assembled to words:

   ```
   CLR            ; zero the accumulators
   SETO 0         ; results at word 0
   LDW  0         ; 2x2 weight tile at byte 0
   LDA  8         ; activations at byte 8
   GEMM 4         ; stream 4 rows
   HALT
   ```

2. `CMD 0x02` (WR_MEM), write the 2ֳ—2 int8 weight tile (row-major), then the
   activation rows.
3. `CMD 0x03` (RUN); poll `uio[6]` DONE.
4. `CMD 0x04` (RD_RES); read `uo`, strobing `uio[1]` to advance ג€” 4 bytes per
   int32 result, little-endian, row-major.

**Honest scope**: this is a teaching/research-scale design ג€” a real ISA and a
real systolic dataflow at 4ֳ—4 int8 scale, not a production accelerator.
